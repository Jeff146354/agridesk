# Agridesk (Academic Document Ledger)

Agridesk is an enterprise-grade Academic Document Workflow System designed to handle the creation, verification, and digital signing of academic letters securely. 

Built with a rigorous **Clean Architecture**, this platform features secure PDF streaming, cryptographic hashing (SHA-256) for document integrity, finite state machine workflows, and a premium modern UI.

## Architecture Overview

Agridesk is split into a decoupled Backend and Frontend:

### 1. [Backend (FastAPI)](./backend/README.md)
A high-performance Python backend implementing Domain-Driven Design (DDD) and Clean Architecture principles.
- **Core Stack**: FastAPI, SQLAlchemy 2.0, PostgreSQL, Alembic, Pytest.
- **Key Features**: 
  - Rich Domain Models with strictly encapsulated Finite State Machines.
  - JWT + bcrypt authentication.
  - SHA-256 document hashing & QR Code generation for anti-tampering.
  - PyPDF & ReportLab for physical coordinate-based signature stamping.
  - 100% Test Coverage on core services (32 tests passing in <6s via in-memory SQLite).

### 2. [Frontend (React + Vite)](./frontend/README.md)
A fluid, modern, and highly responsive web application focused on premium UX.
- **Core Stack**: React 18, Vite, TailwindCSS v4.
- **Key Features**:
  - Drag-and-Drop external PDF wizard (`react-dropzone` & `@dnd-kit/core`).
  - Fluid micro-interactions (`framer-motion`).
  - Secure, in-app PDF rendering bypassing CORS/Download Managers (`react-pdf`).

## Quick Start

### Backend
1. `cd backend`
2. `pip install -r requirements.txt`
3. Setup PostgreSQL and copy `.env.example` to `.env`.
4. `alembic upgrade head`
5. `python -m uvicorn app.main:app --reload`
*Swagger Docs available at: `http://localhost:8000/docs`*

### Frontend
1. `cd frontend`
2. `npm install`
3. `npm run dev`
*App runs at: `http://localhost:5173`*

## Verification System

Documents generated by Agridesk contain a permanently baked-in QR code. Third parties can scan this QR code to be directed to the public verification portal (`/verify/{hash}`), which securely validates the document's SHA-256 signature against the immutable database records without requiring internal system access.
